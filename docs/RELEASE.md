# Release Guide

This app is released as a Windows PyInstaller build wrapped by an Inno Setup installer.

## Version source

The app version source of truth is:

```text
core/version.py
```

Update `APP_VERSION` before tagging a release. Release tags must use the same version with a leading `v`, for example `v0.7.1`.

## Local build

Install dependencies first:

```powershell
python -m pip install -r requirements.txt
```

Build the PyInstaller app:

```powershell
.\scripts\build_exe.ps1
```

Output:

```text
dist\QuanLyHangHoa\QuanLyHangHoa.exe
```

Build the installer:

```powershell
.\scripts\build_installer.ps1
```

Output:

```text
dist\installer\QuanLyHangHoa-Setup-v<version>.exe
```

Build a full local release after creating or choosing the tag:

```powershell
.\scripts\build_release.ps1 -Tag v0.7.1
```

## Version check

Check tag/version consistency:

```powershell
.\scripts\check_version.ps1 -Tag v0.7.1
```

The script fails if:

- the tag does not start with `v`
- `core/version.py` cannot be parsed
- the tag version does not equal `APP_VERSION`

## GitHub release flow

1. Update `APP_VERSION` in `core/version.py`.
2. Commit the version change and any release notes.
3. Create and push the tag:

```powershell
git tag v0.7.1
git push origin v0.7.1
```

GitHub Actions will:

1. install Python dependencies
2. verify the tag matches `APP_VERSION`
3. run tests
4. run `compileall`
5. build the PyInstaller app
6. install Inno Setup with Chocolatey on the Windows runner
7. build the installer
8. create `SHA256SUMS.txt`
9. upload the installer as an Actions artifact
10. publish the installer and checksum to the GitHub Release

## Troubleshooting

### Version mismatch

If release fails during version check, make the tag match `APP_VERSION`.

Example:

```text
APP_VERSION = "0.7.1"
tag = v0.7.1
```

### Inno Setup not found locally

Install Inno Setup 6, add `ISCC.exe` to `PATH`, or set:

```powershell
$env:ISCC_PATH = "C:\Program Files (x86)\Inno Setup 6\ISCC.exe"
```

### PyInstaller failure

Run:

```powershell
python -m pip install -r requirements.txt
.\scripts\build_exe.ps1
```

Check the PyInstaller output under `build\` for detailed errors.
